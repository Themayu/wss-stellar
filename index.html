Principles of Web Server Software

Server Languages

There are numerous languages used in server-side scripting, especially when it comes to scripting websites. The most popular languages are: PHP; ASP.net; and JSP, or “JavaServer Pages.”

Scripts created in these languages are more often than not executed solely on the server and are not interacted with in any way by the client other than through information sent by the client’s web browser to the server when a request is made. They tend to be used to generate HTML markup that can then be displayed in the web page itself, though these languages can be used for many operations on the back-end that aren’t necessarily transparent to the client.

Functionality

Web Server Scripting has many uses on the modern Internet, with its roots in many application types used today. Server-side scripting languages are used on nearly every website you visit today, and power lots of different services, making possible such things as shopping websites and order reservations for restaurants. The software is often used for user profile services, allowing users to sign up to a website and manage their own custom “profile”; allowing expression through a personalised page that allows things such as a custom display name and image to be shown to other users. Other things web server scripting allows is website analysis and upload of files to another system, through normal HTTP or specialised File Transfer Protocol (FTP) server software.
